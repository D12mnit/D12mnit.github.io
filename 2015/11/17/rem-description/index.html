浅谈rem单位的应用

发表于 2015-11-17   |   分类于 前端   |     |   阅读次数

在响应式页面制作的过程中，关于字体大小的设置方面，我一直是使用rem作为单位的，这主要是由于前期在某个网站看到的建议，作为初学者当然就按照建议来了，但是最近在页面调试的时候遇到了一些问题，于是决定来深入的了解一下rem这个单位。

rem是什么？

rem（font size of the root element）是CSS3中新增的相对单位，指的是相对于根元素的字体大小的单位。相比rem，我们可能更熟悉的相对单位是em，也就是相对于父级元素字体大小的单位。那么这两个的不同也可以从定义中看出来，rem相比em也有着自己的优势。

em是相对于父级元素的，那么在实际的使用中给我们的计算带了了很大的不便，除非是通篇使用em。而rem则很好的解决了这个问题，它相对于根元素的字体大小进行设置，可以兼容更多的情况，而且计算简单。在使用中，为了方便计算，将浏览器的根元素字体大小设置为62.5%（也就是10px），这样，1rem = 10px，如：

为何要使用rem？

rem设置字体大小的好处之一是，可以轻易的通过修改根元素的字体大小来修改整个页面的字体大小，比如现在的html{font-size: 20px}，想要改变整个页面的字体大小为原来的两倍，只需改变html{font-size: 40px}即可。这在用户设置浏览器内的字体大小时也同样有效，而使用px则不会有任何改变。

rem的相关问题

目前的各大浏览器对于rem的支持还是比较全面的，具体兼容性可以从can i use网站中查找到。

chrome浏览器问题

目前的chrome浏览器在使用rem作为单位的网页中，初始化页面时，会出现浏览器错误渲染rem单位的问题，导致字体过大。

同时，由于chrome浏览器默认字体大小最小为12px，在将html元素字体大小设置为62.5%时，rem虽然在设置字体大小时使用的是正确的10px，但是如果将rem应用在其他属性如行高，宽度时，rem是以12px计算的，所以会造成偏大的结果，比如设置line-height:10px和line-height:1rem得到的结果是不同的。
