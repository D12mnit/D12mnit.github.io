React学习笔记（一）

发表于 2016-11-28   |   分类于 前端   |   阅读次数

本篇记录了一些React学习中所掌握的基础知识，遇到的一些问题及解决办法。

JSX

学习React就不能离开JSX。使用JSX语法可以在React中使用类XML语法轻松地编写组件，类似于在JS中写HTML，能够比较清晰的展现组建的结构。

我个人理解JSX是将模板引擎融入到JS当中，很多的使用方法和常见的模板引擎语法类似。

组件间的通信

React是单项数据流的，组件间的信息传递基本上有以下几种：

  • 父组件向子组件传递
  • 子组件向父组件传递
  • 兄弟组件之间的传递

首先说说父组件向子组件传递信息，这个最为常见，也比较简单，通过props来传递。

而子组件向父组件传递，这个一般是通过父组件定义一个回调函数，通过props传递给子组件，当子组件状态发生改变需要反映给父组件时，通过调用props传递的回调函数，改变父组件状态即可。不过如果层级嵌套过多，就会出现一层一层的向上回调的情况，用props的回调函数来处理就会比较复杂。

由于目前我自己做的项目不是很复杂，所以并没有遇到层级过深的情况。不过有读过一些相关的文章，可以通过设置组件的context来避免props的层层传递（react-redux便是通过将state注入context实现的）。但是官方文档中也不推荐新手尝试使用context。。。

最后一种兄弟组件的通信的话，一般是通过他们的共同父组件进行数据传递。或者通过定义一个全局的自定义事件来触发数据变更。

组件生命周期

组件的生命周期，我个人是参照着官方文档的描述来记忆的，比较清晰，分别是mounting，updating，unmounting三个阶段。除unmounting阶段，每个阶段都会经历will,render,did三个小阶段。

以下均是es6语法下的生命周期。

mounting阶段是组件生成并插入dom中时所发生的阶段。发生顺序为：

  • constructor()
  • componentWillMount()
  • render()
  • componentdidMount()

首先是constructor()，es6语法下组件类继承自React.Component类，在constructor构造函数中进行组件的props获取，初始化组件state

componentWillMount()会在render()前执行，在其中对state的操作不会引起re-render。这个过程好像是和server端的render有关，目前我还没有用过。

然后是render()函数，会生成组件dom结构并插入。官方说明render()应为纯函数，不应改变组件的state等，使得相同的传入有相同的返回。同时该阶段无法获取真实的dom结构，对真实dom的相关操作应在componentDidMount()中或其后的生命周期函数中。

componentDidMount()在render()后执行，此时dom结点已经插入，可以对dom进行事件绑定，绑定计时器等。同时ajax获取数据等也应该在该函数中。

下面是updating阶段，是当组件的state改变或者接收到新的props时，会更新组件。

  • componentWillReceiveProps()
  • shouldComponentUpdate()
  • componentWillUpdate()
  • render()
  • componentDidUpdate()

componentWillReceiveProps()阶段发生在组件接收到新props时，如果组建的state与props有关，则在这个函数中计算并更新state。如果只是组件内部改变state不会触发该函数。

shouldComponentUpdate()在接收到新的props和state时，决定是否触发render()更新组件，默认返回true。可以通过重该函数来优化性能(purerender就是重写了该函数)。如果返回false，则componentWillUpdate()，render()，componentDidUpdate()都不会触发。

componentWillUpdate()会在接收到新的props和state后触发render()前立即调用，可以做一些处理，但此时无法调用this.setState()来更新state。

componentDidUpdate()类似componentDidMount()，只是在更新时的render()后立即执行。

最后是unmounting阶段，只有一个componentWillUnmount()函数，会在组件即将从DOM中卸载前执行，通常用来取消事件绑定，清除计时器，清除手动添加的dom结构等。

生命周期官方的描述十分清晰，个人在项目中的使用，觉得只要理解他们的作用就一般不会有太大问题，不过也可能是我个人做的小项目还遇不到会发生的问题。最主要的可能就是要注意在componentDidMount中绑定的一些事件与计时器要记得在componentWillUnmount中清除掉吧。其次就是在shouldComponentUpdate()中做一些state的前后判断来优化一下性能，不过我的项目不大，优化了感觉效果基本看不出来。。。

其他

记录一些零碎的东西吧。

es6语法：

es6自己接触挺久了，自己的项目一直是用的es6，很多新特性用起来都很友好，不过需要babel的支持。

react中的es6语法也很清晰，很多新特性都能用上，比如通过箭头函数来绑定this到组件，通过object spread语法(需要安装babel的stage-2阶段来支持)来给深层级组件传递props(<My {...props} />)(不过文档里不推荐经常使用，会传递一些不需要的props)。还有通过解构语法把props赋给其他变量等。

mobile方面：

移动端方面，由于之前的实习主要做的是移动端的H5页面，了解的多为交互上的内容，比如移动端的touch事件，css动画，手势等。其他的目前还在学习阶段吧。
